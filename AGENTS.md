Angular Coding Agent Instructions

1. Always use proper Angular components for UI implementation. Break UI into reusable, maintainable components where appropriate.

2. Use Tailwind CSS as the first choice for styling. For complex animations, advanced layouts, or styles that Tailwind cannot handle cleanly, use custom CSS.

3. Keep the code clean, readable, and not overly complex. Prefer simple, maintainable solutions over clever or unnecessary abstractions.

4. Avoid excessive comments. Add only short one-line comments when the logic is complex or not immediately obvious.

5. Before starting implementation, always refer to the latest official Angular, Tailwind CSS, and related library documentation.

6. If anything is unclear, blocked, or requires assumptions, ask the user or research the correct approach before implementing. Do not guess or implement based on unsupported assumptions.

7. For every new task, first do proper research and analysis:
   - Understand the requested feature or change.
   - Review the existing codebase.
   - Check how the UI should look and behave.
   - Identify where the change fits best.
   - Determine what files, components, services, styles, or tests need updates.

8. Ensure the final implementation fits naturally into the existing project structure, coding style, and UI patterns.

9. Do not introduce unnecessary dependencies, complex patterns, or large refactors unless they are clearly required.

10. Keep accessibility, responsiveness, and performance in mind for all UI work.
